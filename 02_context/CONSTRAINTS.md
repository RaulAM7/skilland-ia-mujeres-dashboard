## Tooling Limits
- Current workspace is docs-first according to `01_harness/STACK.md`; implementation stack must be recorded there once code is introduced.
- Work must follow the harness: raw inputs in `00_inbox/`, distilled context in `02_context/`, one active spec in `03_specs/now/`, deliverables in `04_outputs/`, scratch in `05_scratch/`.
- Existing context must stay readable in roughly 5 minutes; do not copy raw inbox dumps into context files.
- Target app stack from sources: React, Vite, TypeScript, shadcn/ui, shadcn-admin, Tailwind, Vercel Functions, server-side CRM/Twenty client.
- Dashboard runtime should query CRM/Twenty through dashboard server API, not from the browser.
- Local development must support mock data without CRM access.

## Non-Negotiables
- Do not invent facts; missing data must be marked `Unknown`.
- CRM/Twenty is the runtime source of truth for funnel data.
- The snapshot is owned and generated by the dashboard API.
- Do not expose CRM tokens in frontend code or `VITE_*` variables.
- Do not commit real `.env` values.
- MVP is read-only: no CRM edits, no email sending, no draft creation, no stage changes, no destructive actions.
- Do not touch `skilland-crm` initially unless a critical metric or field gap requires it.
- Do not include unnecessary personal data, full email bodies or secrets in snapshots/logs.
- Dashboard must not be publicly open if it contains internal funnel or personal data.
- Unknown CRM stages must not break the UI; they should map safely and generate warnings.

## Tone and Brand Constraints
- Serious, institutional, clear, ambitious, impact-oriented and pedagogical.
- Avoid paternalistic, condescending, hype-heavy or "course basic for women" framing.
- Position IA Mujeres as measurable women-in-AI empowerment programs with platform, itineraries, microcredentials and impact reporting.
- Funnel tone is "institutional conversation first", not aggressive automation or direct cold sale.
- Dashboard UX should be operational, executive, readable and action-focused: a command center, not a vanity BI page or duplicated CRM.
